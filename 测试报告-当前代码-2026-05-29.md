## 新发现 Bug

### BUG-A 🔴 _triggerCloudRecovery 无条件关闭 cloudWs

**文件**：`main.js` `_triggerCloudRecovery` 函数  
**严重度**：高 — 影响所有云连接手机

```javascript
function _triggerCloudRecovery() {
  // ...
  if (cloudWs) {
    cloudWs.close();     // ← 即使cloudWs已正常连接也关闭！
    cloudWs = null;
  }
  // 重建...
}
```

**场景**：PC 有 2 台手机通过云端正常连接。用户用插件点拨号给手机 C（C 未连接）。`_triggerCloudRecovery` 触发 → 关闭 cloudWs → **两台正在使用的手机同时掉线** → 5 秒恢复。

**后果**：拨号失败一次就让所有云连接手机断线重连。

**应改为**：只在 `cloudWs` 未连接（readyState !== 1）时才重建。

---

### BUG-B 🟡 _triggerCloudRecovery 与旧 close handler 竞态

**文件**：`main.js`  
**严重度**：中 — 可能重复连接

```
_triggerCloudRecovery:
  1. cloudWs.close() → async
  2. 旧 close handler → generation匹配 → _scheduleCloudReconnect → 设置定时器
  3. connectCloudServersFromList → _cloudTraversalGeneration++ → 立即连接
  4. (若干秒后步骤2的定时器触发) → 再次 connectCloudServersFromList
     → connectCloudServer: cloudWs已连接 → close → 重建 → 第二次掉线
```

**应改为**：`_triggerCloudRecovery` 里先 `clearTimeout(cloudReconnectTimer)`。

---
